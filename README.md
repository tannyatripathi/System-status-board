# System-status-board
A lightweight, zero-dependency service health dashboard designed for manual status tracking and operational visibility.

Service Status Board
Why I Built This
In Service Operations, reliability is everything. I built this status dashboard to show how a team can maintain visibility over their systems without needing complex or fragile infrastructure. It is designed to be a lightweight, "zero-failure" tool for manual status tracking.

How it Works
This project was a study in Pure HTML and CSS. I intentionally avoided JavaScript to ensure the tool remains functional even in environments where scripts might be restricted or prone to error.

Interactive UI: I used a "hidden checkbox" CSS technique. When a user clicks a service status, the styling updates instantly from "Offline" (Red) to "Operational" (Green).

Clean Design: The focus was on creating an "Enterprise-grade" look that is easy for stakeholders to read at a glance.

Core Competencies
Front-End Layouts: Structuring data clearly using HTML5.

Advanced CSS: Utilizing pseudo-elements and sibling selectors for interactivity.

Operational Thinking: Designing tools that prioritize simplicity and uptime.
